# Proposal: 统一归档页面与灵玉界面风格

## Summary

将归档页面入口迁移至设置面板的"开发者"分组，同时将归档页面和灵玉明细界面重构为与设置子页面一致的全屏右滑风格，提升应用整体的 UI 一致性。

## Motivation

当前存在以下问题：

1. **归档入口位置不合理**：归档按钮放在主页面顶部，但归档是低频操作，占用了宝贵的顶部空间
2. **归档页面风格不统一**：归档页面使用独立的头部和筛选栏风格，与设置子页面风格差异较大
3. **灵玉界面风格不统一**：灵玉明细使用居中小弹窗，与其他全屏页面交互方式不一致

## Goals

- 归档入口移至设置面板 > 开发者分组
- 归档页面采用 SubPageLayout 布局（头图 + 标题 + 内容）
- 归档页面筛选功能改为标签管理风格的 Tab 切换
- 灵玉界面改为全屏右滑页面，使用 SubPageLayout 布局
- 保持灵玉入口在原位（顶部灵玉图标）

## Non-Goals

- 不改变归档页面的核心功能逻辑
- 不改变灵玉数据的计算逻辑
- 不增加新的功能，仅做 UI 统一

## Scope

### 涉及的 Specs

1. **settings-panel** - 添加归档入口到开发者分组
2. **spirit-jade-system** - 修改灵玉明细界面展示方式

### 涉及的文件

- `src/pages/dc/index.tsx` - 移除顶部归档按钮，修改灵玉点击逻辑
- `src/pages/dc/panels/archive/` - 归档页面重构
- `src/pages/dc/panels/settings/` - 添加归档入口和归档子页面
- `src/pages/dc/components/SpiritJadeHistoryPopup/` - 重构为全屏页面

## Risks

- 用户习惯变化：归档入口位置改变可能需要用户适应
- 缓解措施：设置面板入口明显，用户容易找到

## Timeline

预计 1-2 天完成
